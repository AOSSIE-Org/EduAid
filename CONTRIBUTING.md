# Contributing to EduAid

Thank you for your interest in contributing to EduAid! We welcome all kinds of contributions.

## How to Contribute

- **Raise Issues**: If you find a bug or have a feature request, please open an issue on GitHub.
- **Fix Issues**: Check open issues and submit a pull request (PR) with your fix.
- **Improve Documentation**: PRs to improve documentation are always welcome.
- **Spread the Word**: Talk about AOSSIE and EduAid in your community!

## Setting Up the Project Locally

1. **Clone the Repository**
   ```bash
   git clone https://github.com/AOSSIE-Org/EduAid.git
   cd EduAid
   ```

2. **Backend Setup**
   - See the [README.md](README.md) for detailed backend setup instructions.

3. **Web App Setup**
   - Navigate to `eduaid_web` and run:
     ```bash
     npm install
     npm run start
     ```

4. **Desktop App Setup**
   - Navigate to `eduaid_desktop` and run:
     ```bash
     npm install
     npm run dev
     ```

## Proposing Changes

- Fork the repository and create a new branch for your feature or bugfix.
- Make your changes and test them locally.
- Submit a pull request with a clear description of your changes.

## Pull Request Guidelines

- Reference related issues in your PR description.
- Ensure your code follows the existing style.
- Write clear, concise commit messages.
- Add tests if applicable.

## Coding Style

- Use meaningful variable and function names.
- Keep functions small and focused.
- Write comments where necessary.

## Commit Message Format

- Use the present tense (“Add feature” not “Added feature”).
- Use imperative mood (“Fix bug” not “Fixed bug”).
- Reference issues where relevant (e.g., “Fix #123”).

Thank you for contributing!
